# markdown学习笔记

## markdown标题

- ### 使用 = 和 - 标记一级和二级标题

  eg：

```
一级标题
=======
二级标题
-------
```

- ### 使用#号标记

  使用 **#** 号可表示 1-6 级标题，一级标题对应一个 **#** 号，二级标题对应两个 **#** 号，以此类推。

  eg：

  ```
  # 一级标题
  ## 二级标题
  ### 三级标题
  #### 四级标题
  ##### 五级标题
  ###### 六级标题
  ```

- ### 注意事项：

  1. **# **号和标题文字之间需要有一个空格

  eg：

  ```
  # 正确
  #错误
  ```

  2. **#**号必须在行首，前面不能有任何的字符。

  3. 一个文档通常只使用一个一级标题作为文档的主标题。

  4. 标题简洁明了，一般不超过 10 个汉字或 20 个英文字符

  5. 使用描述性词语，避免模糊的标题如"其他"、"杂项"

- ### 嵌套结构

  标题的层次结构应该遵循逻辑顺序，不应该跳级使用。

  eg:

  ```
  推荐：
  # 主题
  ## 二级标题
  ### 三级标题
  避免：
  # 主题
  ### 三级标题
  ## 二级标题
  ```

## markdown文本格式

- #### 字体

  1. 粗体

     使用两个星号 ******或两个下划线 **__** 包围文字。

     eg：

     ```
     **粗体**
     __粗体__
     ```

  2. 斜体

     使用一个星号 ***** 或一个下划线 **_** 包围文字。

     eg:

     ```
     *斜体*
     _斜体_
     ```

  3. 粗斜体

     使用三个星号 ***\**** 或三个下划线 **___**。

     eg：

     ```
     ***粗斜体***
     ___粗斜体___
     ```

  4. 文本高亮

     使用两个==。

     eg：

     ```
     你好 ==hello==
     ```

  5. 建议

     - 推荐使用星号 `*` ，因为星号在各种 Markdown 解析器中兼容性更好.
     - 不要过度使用强调，重点突出才有效果。
     - 在中英文混合时，建议在强调符号前后加空格以提高可读性

- #### 线

  1. 分割线

     用三个以上的星号、减号、底线来建立一个分隔线，行内不能有其他东西，也可以在星号或是减号中间插入空格。

     eg：

     ```
     ***
     ---
     * * *
     *****
     ------
     ```

  2. 删除线（~~hello~~）

     在文字的两端加上两个波浪线 **~~** 。

     eg：

     ```
     ~~hello~~
     ```

  3. 下划线（<u>hello</u>）

     可以通过 HTML 的 **<u>** 标签来实现。

     eg：

     ```
     <u>hello</u>
     ```

- #### 脚注

  脚注是对文本的补充说明。

  eg:

  ```
  文本[^!]。
  [^!]:注明的内容
  ```

- #### 行内代码标记

  行内代码用于在正文中标记代码片段、命令、变量名等   

  1. 基本语法

     使用一个反引号 **`** 包围代码[^!]

     eg：

     ```
     `hello`
     ```

     [^!]:当代码本身包含反引号时，使用两个反引号包围。eg:

     ```
     ```hello```
     ```

  2. 应用场景

  - 技术文档中的 API 名称、函数名

  - 配置文件中的参数名

  - 命令行指令

  - 键盘快捷键（如 `Ctrl+C`）  

- #### 换行与段落

  1. 换行

     - 普通换行：

       段落的换行是使用两个以上空格加上回车。

     - 强制换行

       - 行尾两个空格
       - HTML 换行标签
       - 反斜杠\

  2. 段落

     段落是文本的基本单位

     - 基本规则

       - 段落由一个或多个连续的文本行组成
       - 段落之间由一个或多个空行分隔
       - 普通段落不应该用空格或制表符缩进

     - 正确写法

       eg：

       ```
       段落1。
       
       段落2。（注意上面有一个空行分隔）
       
       段落3。
       ```

  3. 空行的作用

     - 分隔段落
     - 分隔不同元素

  4. 实践建议

     - 在标题和内容之间留空行
     - 在列表前后留空行
     - 在代码块前后留空行
     - 保持一致的空行使用习惯

## markdown的列表

- #### 无序列表

  1. 使用星号(*****)、加号(**+**)或是减号(**-**)作为列表标记，这些标记后面要添加一个空格，然后再填写内容。

     eg：

     ```
     - 1
     - 2
     - 3
     ```

  2. 选择建议

     - 建议统一使用减号 **-**，因为它在视觉上更清晰
     - 在同一文档中保持一致的标记方式
     - 标记符号后必须有一个空格  

- #### 有序列表

  1. 使用数字并加上 **.** 号来表示.

     eg：

     ```
     1. 第一项
     2. 第二项
     3. 第三项
     ```

  2. 注意：

     - 数字可以不连续
     - 可以从指定数字开始  

- #### 列表嵌套

  列表可以嵌套使用，创建多层次的结构。

  1. 无序

     eg：

     ```
     -A
      -a
       -1
       -2
      -b
     ```

  2. 有序

     eg：

     ```
     1. A
        1. a
           1. xxx
           2.xxx
        2. xxx
     ```

  3. 混合

     eg：

     ```
     1. xxx
        - xxx
        - xxx
          1. xxx
          2. xxx
        - xxx
     2. xxx
     ```

  4. 规则

     - 子列表需要缩进 2-4 个空格（推荐 2 个）
     - 保持一致的缩进长度
     - 可以无限层嵌套，但实际使用中建议不超过 3 层  

- #### 任务列表

     任务列表是 GitHub 风格 Markdown 的扩展功能。

  1. 基本语法：

     eg:

     ```
     - [ ] 未完成的任务
     - [x] 已完成的任务
     - [ ] 另一个未完成的任务
     ```

  2. 使用技巧：

     - 方括号内的空格和 x 很重要：`[ ]` 和 `[x]。`

     - 可以与嵌套列表结合使用。

     - 在项目管理、学习计划、生活清单中特别有用。

     - 某些编辑器支持点击复选框来切换状态。

     ## markdown引用块

     引用块用于突出显示重要信息、引用他人观点或创建视觉层次

     1. 单级引用

        - 基本语法

           区块引用是在段落开头使用 **>** 符号 ，然后后面紧跟一个空格符号

          eg:

          ```
          > hello
          ```

        - 多行引用[^!]

          eg:

          ```
          > 段落1的第一行
          > 段落1的第二行
          >
          > 段落2的第一行
          ```

          [^!]:简化写法：只在第一行使用 **>** ，其余行会自动包含在引用中。

     2. 多级嵌套引用

        另外区块是可以嵌套的，一个 **>** 符号是最外层，两个 **>** 符号是第一层嵌套。

        eg：

        ```
        > 最外层
        > > 第一层嵌套
        > > > 第二层嵌套
        ```

     3. 区块

        如果要在列表项目内放进区块，那么就需要在 **>** 前添加四个空格的缩进

        eg：

        ```
        - 第一项
            > 1
            > 2
        - 第二项
        ```

     4. 引用块内的其他元素
        - 标题
        - 列表
        - 代码
        - 链接与图片

  ## marldown的代码

- 行内代码

  如果是段落上的一个函数或片段的代码可以用反引号把它包起来（**`**）

  eg：

  ```
  `printf`函数
  ```

  1. 常见用法

     - 函数名：使用 `console.log()` 输出信息
     - 变量名：将值赋给 `userName` 变量
     - 命令行：运行 `npm install` 安装依赖
     - 键盘按键：按 `Ctrl+C` 复制内容
     - 文件名：编辑 `index.html` 文件

  2. 特殊字符转义

     当需要在行内代码中显示反引号或其他特殊字符时，需要进行转义处理。

     - 使用双反引号包围单反引号
     - 使用多个反引号包围
     - 其他特殊字符处理
       1. HTML 标签：`<div>` 元素
       2. 数学符号：计算 `x + y = z`
       3. 特殊符号：使用 ` ` 表示空格

- 代码区块

  1. 缩进式代码块

     代码区块使用 4 个空格或者一个制表符（Tab 键）。

     - 语法格式

     - eg：

       ```
       文本
       
           这是缩进式代码块
           每行前面有四个空格
           保持代码的原始格式
           
       继续文本
       ```

  2. 三反引号代码块

     可以用 **```** 包裹一段代码，并指定一种语言（也可以不指定）

     eg：

     ````
     ```
     多行代码内容
     可以包含空行
     保持原有缩进
     ```
     ````

  3. 注意事项：

     - 缩进式代码块前后需要空行分隔
     - 所有代码行必须保持一致的缩进
     - 不支持语法高亮
     - 在列表中使用时需要8个空格缩进

## markdown链接

链接是使 Markdown 文档具有交互性的关键元素。链接语法能让你创建内容丰富、更易于导航的文档。

- 使用方法

  eg：

  ```
  [链接名称](链接地址)
  [链接文字](链接地址 "可选的标题")
  or
  <链接地址>
  ```

- 链接标题的作用
  - 当鼠标悬停在链接上时显示提示信息
  - 对搜索引擎优化和无障碍访问有帮助
  - 标题文字放在双引号、单引号或括号中都可以

- 参考链接

  参考式链接将链接定义与使用分离，让文档更整洁，特别适合长文档或需要多次引用相同链接的情况。

  - 基本语法

    eg：

    ```
    markdown[链接文字][参考标签]
    
    [参考标签]: URL "可选标题"
    ```

    eg：

    ```
    这个链接用 1 作为网址变量 [Google][1]
    这个链接用 runoob 作为网址变量 [Runoob][runoob]
    然后在文档的结尾为变量赋值（网址）
    
      [1]: http://www.google.com/
      [runoob]: http://www.runoob.com/
    ```

    结果如下：

    这个链接用 1 作为网址变量 [Google][1]
    这个链接用 runoob 作为网址变量 [Runoob][runoob]
    然后在文档的结尾为变量赋值（网址）

    [1]: http://www.google.com/
    [runoob]: http://www.runoob.com/

  - 简化写法

    当参考标签与链接文字相同时，可以省略第二个方括号

    eg：

    ```
    markdown 我喜欢使用 [GitHub][] 来管理代码。
    
    [GitHub]: https://github.com
    ```

    

- 参考链接的优势
  - 文档正文更清爽，不被长 URL 打断
  - 便于链接的统一管理和更新
  - 相同链接可以重复使用，避免重复定义
  - 链接定义可以放在文档任意位置（通常放在末尾）

- 组织技巧

  - 自动连接识别

    现代 Markdown 解析器通常支持自动识别 URL 和邮箱地址

    1. URL 自动识别：

       eg：

       ```
       markdown直接输入网址：https://www.example.com
       用尖括号包围：<https://www.example.com>
       ```

    2. 邮箱自动识别：

       ```
       markdown联系邮箱：example@email.com
       或者：<example@email.com>
       ```

  - 注意事项

    - 自动识别功能依赖于具体的 Markdown 解析器
    - 为了确保兼容性，建议使用标准的链接语法
    - 某些特殊字符可能影响自动识别

- 锚点链接的使用

  锚点链接用于在同一文档内跳转，特别适合长文档的导航

  - 跳转标题

    ```
    ## 目录
    - [第一章：介绍](#第一章介绍)
    - [第二章：安装](#第二章安装)
    
    # 第一章：介绍
    这里是介绍内容...
    
    # 第二章：安装
    这里是安装说明...
    ```

  - 锚点规则

    1. 标题会自动生成锚点

    2. 锚点名称通常是标题的小写形式

    3. 空格替换为连字符

    4. 移除特殊字符

  - 手动创建锚点

    eg:

    ```
    <a id="custom-anchor"></a>
    ## 自定义锚点位置
    
    [跳转到自定义位置](#custom-anchor)
    ```

    eg：！页面顶部返回链接

    ```
    [回到顶部](#)
    ```
