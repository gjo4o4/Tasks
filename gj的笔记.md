## **Geek&Geek精神** 

### **Geek**

- 可以翻译为笨蛋、奇葩、怪才、对…痴迷狂热的人（极客）。最早指在19世纪是马戏团中吃虫子的怪人，但到了20世纪后半叶，它逐渐的褒义化，演变为指对某个领域极度痴迷、专业到近乎"狂热"的人-﹣比如科技宅、电影迷、动漫控、健身狂热者。我觉得geek更像是兴趣狂热者，是充满了好奇与热爱的人。是会为了兴趣研究得很深，为了技术精进得很细。他们不止满足于知道，而对理解更感兴趣。  

### **Geek精神**
- 极致的热爱与专注：

  - 对领域发自内心热爱，追求深度与细节。 

- 好奇与探索欲

  - 不满足于知道，主动探索未知。

- 不循规蹈矩

  - 不随波逐流，追求本质，敢于质疑，专注自我。

- 开源与分享

  - 乐于共享知识、代码、方案，相信社区协作推动进步。

- 力求完美

  - 追求逻辑、效率，化繁为简，力求完美。

  > 总之我认为geek精神就是一种对技术又或者其他自己热爱的事物有着极致热爱与探索欲，愿意深度钻研、追求本质、专注自我的一种精神。

## **GitHub**
###  Github

> GitHub 是基于 Git 的全球最大代码托管与协作平台。它让全世界的开发者可以方便地存储、分享和协作开发软件项目。

- 代码仓库
  - GitHub 是一个安全稳定的代码存储仓库，可以保存项目代码和所有历史版本，方便进行版本管理和回溯。

- 协作平台
  - GitHub 可以实现高效的团队协作，支持多人共同开发项目，通过分支、合并等功能让协作更规范、更高效。

- 开源社区
  - GitHub 汇聚了大量优秀开源项目，供所有人学习、使用和共同改进。  

### Fork&Pull Request

- #### Fork（分支与复刻）作用：

1. 复制副本

2. 独立开发学习

3. 项目备份  

- #### pull Request（拉取请求）作用：[^ps]

1. 代码审查

2. 促进团队协作与讨论

3. 保障代码质量
   [^ps]： 如果想修复某个开源项目的 Bug 或添加新功能，但没有该项目的直接写入权限，你可以先 Fork 它。修改完成后，你可以通过 Pull Request（拉取请求） 功能，向原作者提交你的修改建议。如果原作者认可，你的代码就会被合并到原项目中 。
