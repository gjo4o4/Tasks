# 页面编后感

1. 出现的问题

- 上传

  1. 第一次上传还是用的GitHub直接上传

  2. 用VScode的上传后，发现传到了master分支

     从VScode上传之后，在Github网站阶段三main分支并没有找到gj_website的文件，而在master文件里。尝试在VScode上改变分支上传，发现行不通。之后在终端修改，并删除了master分支。

  3. images文件夹上传失误散开

     从VScode推送至Github上图片效果没有渲染出来，因为没有把images文件夹一起推送。

  4. 网页链接出不来，出来的是README.md的网站

     因为index/about.html 不在仓库的根目录下，所以Github Pages优先加载了README.md页面展示。而网页文件在阶段3这个子文件里。

     网页链接为[gj的个人主页](https://gjo4o4.github.io/Tasks/阶段3/index.html)

  5. 修改后忘记用VScode推送至Github

- 代码

  1. 图片渲染
  
     images文件夹没有正确上传至Github，导致图片没有渲染出来。
  
  2. 效果“张冠李戴”
  
     爱好板块与图片板块的效果出不来，或者错误。
  
  3. 导航栏抱团
  
     头像，个人主页，以及首页与about me缩在一起。
  
  4. 文字与背景的颜色调试错误
  
     文字与背景过于相似，导致不清晰。

2. 感受

   阶段3的任务是我首次编写网页的尝试，是一次将理论落地的实践，也更进一步认识。每一步的尝试有可能出现正确的效果，也有可能出现第一次见到的问题。

   上传路径不对，网页“失踪“，图片路径不对，渲染不出来，样式错误，结构不完整，效果不对或者出不来...等等问题。不仅代码要是对的，GitHub上的部署逻辑也很重要。解决每一个问题的过程都让我对HTML的认识更进一步。

   这次编写让我意识到一个问题，实践练习非常重要，过程的每一步都影响着最终结果，不能只停留在理论知识。
