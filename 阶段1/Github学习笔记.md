# Github学习笔记

##  基础入门

- 创建并使用仓库

  仓库会把属于你正在做的“项目”或项目的项目归组起来。通常，仓库会包含一个`README`文件，即包含你项目信息的文件。`README `文件使用 Markdown 编写，这是一种易于阅读、易于编写的纯文本格式化语言。

  1. 在任意页面的右上角，选择 ，然后点击“新建仓库（New repository）”。
  2. 在“仓库名称”框中输入要取的仓库名。
  3. 在“描述”框中输入简短描述。
  4. 选择你的仓库是公开还是私有。
  5. 选择添加`README`文件。
  6. 点击创建仓库。

- 开创新分支

  分支让你同时拥有不同版本的仓库。你想为项目添加新功能而不改变主要代码源时，分支很有用。在不同分支上完成的工作，直到合并主分支后才会显示出来。

  1. 点击你仓库的`code`标签。
  2. 文件列表上方，点击`main`下拉菜单。
  3. 在文本框中输入分支名称。
  4. 点击`eate branch`：`main`的 `readme-edits`

- 修改与提交

  你可以对仓库里的文件进行修改和保存。在GitHub上，保存的更改被称为提交。每个提交都有一个对应的提交消息，说明为何做出了某个特定更改。提交消息记录你更改的历史，让其他贡献者理解你做了什么以及原因。

  1. 在你创建的分支下，点击文件。
  2. 要编辑文件，请点击右上角的铅笔键。
  3. 在编辑器里，写一些关于你自己的介绍。
  4. 点击`Commit changes`。
  5. 在`Commit changes`框中，写一条提交消息描述你的更改。
  6. 点击`Commit changes`。

- 打开&合并拉取请求

  拉取请求是GitHub协作的核心。当你发起拉取请求时，你是在提议你的更改，并请求有人审核并拉入你的贡献，并将其合并到他们的分支中。拉取请求显示了两个分支内容的差异。变化、添加和减法以不同颜色显示。

  - 打开

  1. 点击你仓库的`Pull requests`标签。
  2. 点击`New pull request`.
  3. 在`Example Comparisons`框中，选择你创建的分支 ，与（原始分支）进行比较。
  4. 查看比较页面上的差异，确保这些是你想提交的。
  5. 点击`Create pull request`。
  6. 给你的拉取请求起个标题，并简要描述你的更改。你可以添加表情符号、拖拽图片和动图。
  7. 点击`Create pull request`。

  - 审查

    当你开始与他人合作时，这正是你请求他们评价的时候。这样你的协作者可以在你合并修改到分支之前，对你的拉取请求提出评论或提出修改建议。

  - 合并

    当拉取请求存在冲突时，GitHub 会提醒你，并在冲突解决前阻止合并。你可以提交一个提交来解决冲突，或者在拉取请求中使用注释与团队成员讨论冲突。

    1. 在拉取请求底部，点击`Merge pull request`以将变更合并为 。
    2. 点击`Confirm merge`。您将收到请求成功合并并关闭的消息。
    3. 点击`Delete branch`。现在你的拉取请求已经合并，且更改已开启，你可以安全地删除该分支。如果你想对项目做更多修改，可以新建分支并重复这个过程。
    4. 点击回到你仓库的`code`标签页，查看你发布的更改。

## 如何在GitHub上寻找灵感

GitHub 是一个庞大的开源社区。你可以在GitHub上探索有趣的仓库、主题、代码、人物和组织，这些都能激发你的创作灵感，或支持你的学习。

对于感兴趣的内容，可以：

1. 给仓库或主题加上星号，方便你以后轻松找到。

2. 关注个人或组织，这样你就能随时了解他们的动态。

3. 下载有用的仓库或代码，并根据自己的需求进行定制。
4. 通过打开拉取请求，为其他用户的项目做出贡献。

- 访问Explore Github
  1. 浏览[探索GitHub](https://github.com/explore)。
  2. 浏览热门资源库和主题。
  3. 点击`star`旁边是你感兴趣的资料库和主题，这样你以后可以轻松找到它们。
  4. 进入你的[星级页面](https://github.com/stars)，查看所有星级仓库和主题。

- 搜索某个主题&题目

  1. 在搜索栏输入关键词或查询。

  2. 使用左侧边栏筛选结果。

  3. 给符合你兴趣的仓库做星号

## 下载文件

GitHub.com 拥有数百万个开源软件项目，你可以复制、定制并用于自己的目的

- 获取文件的方法
  1. 将仓库文件的`下载(Downlload)`为压缩包到你自己的（本地）电脑上。
  2. 用 `克隆（Clone）`一个仓库到你的本地电脑。
  3. `分支（Fork）`一个仓库，在GitHub上创建一个新的仓库。

- 概念区分
  1. 下载
     - 定义：将仓库文件保存到本地电脑。
     - 适用背景：使用或自定义文件内容，但又不想应用版本控制。
  2. 克隆
     - 定义：制作仓库数据的完整副本，包括每个文件和文件夹的所有版本。
     - 适用背景：需要在本地电脑上完整保存仓库，使用 Git 来跟踪和管理你的更改。
  3. 分叉
     - 定义：在 GitHub 上创建一个新仓库，链接到你的个人账户，并与原仓库共享代码和可见性设置。
     - 适用背景：想用原始仓库的数据作为你自己在 GitHub 上项目的基础。或者，想用分支来对原始仓库提出修改建议。分叉仓库后，你可能还想克隆仓库，这样可以在本地电脑上处理更改。

- 下载方法

  1. 找到到要下载的文件。

  2. 在文件列表上方，点击`code`。

  3. 下载。

## 如何将文件上传到Github

- 上传的作用：
  1. 编辑文件时应用版本控制，确保项目历史受保护且易于管理。
  2. 备份你的作品，因为你的文件现在存储在云端。
  3. 将该仓库置顶到你的个人资料中，这样其他人也能看到你的作品。
  4. 与他人分享和讨论你的作品，无论是公开还是私下。

- 上传的步骤：

  1. 新建一个仓库或者使用原有仓库。
  2. 将文件上传到仓库。
     - 在页面右侧，选择`Add file`下拉菜单。
     - 在下拉菜单中，点击`Upload files`。
     - 在你的电脑上，打开包含你作品的文件夹，然后将所有文件和文件夹拖拽到浏览器中。
     - 在页面底部，在`Commit changes`下，选择“直接提交到分支，然后点击`Commit changes`提交更改。

  3. 编辑仓库中的`README`文件。
